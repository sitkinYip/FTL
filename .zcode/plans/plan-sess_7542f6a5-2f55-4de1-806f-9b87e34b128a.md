# FTL Tauri + TS(Vue)重写计划

## 决策摘要(已确认)
| 项 | 选择 |
|---|---|
| 前端 | **Vue 3 + naive-ui**(组合式 API + Vite) |
| 桌面壳 | **Tauri v2** |
| 后端 | **Python sidecar**(fontTools 无等价实现,复用现有逻辑) |
| 打包 | **先系统 Python,后 PyInstaller 打包**(分两阶段) |
| 平台 | **macOS + Windows + Linux 三端** |
| 仓库 | **全新仓库**(当前 `feature/tauri-ts-rewrite` 分支,目录 `ftl-tauri/`) |

## 关键技术结论(调研所得)
1. **后端不可重写**:fontTools 的 subsetter(GPOS/GSUB 特性过滤、字形闭包、name 表、CFF desubroutinize)无等价纯 Rust/JS 库,重写不现实。现有 Python 后端 ~400 行零 UI 耦合,直接复用。
2. **文件拖拽走 Tauri native API**:不用 HTML5 `dataTransfer.files`(拿不到绝对路径),而用 `getCurrent().onDragDropEvent()` 拿原生文件路径——这正是桌面端子集化需要的。naive-ui 的 Upload 用于点击选择 + 视觉呈现。
3. **通信协议**:sidecar 用 **NDJSON over stdio**(每行一个 JSON 消息),Tauri `Command::new_sidecar()` 逐行读 stdout、写 stdin,天然适配。
4. **「常用中文」预设有 bug**:只有几百字且带"夸克网盘"前缀,移植时顺带修复(内嵌真实 GB2312 常用字表)。

## 通信协议(NDJSON over stdio)

**前端 → sidecar(stdin,一行 JSON)**:
```jsonc
// 开始处理
{"type":"process","id":"batch-001","files":["/a.ttf","/b.otf"],"characters":"abc","formats":["woff2","ttf"],"options":{"keep_layout":true,"keep_names":true,"notdef_glyph":true,"glyph_names":false,"keep_hinting":false},"output_mode":"subdir","custom_dir":null}
// 取消
{"type":"cancel","id":"batch-001"}
```

**sidecar → 前端(stdout,每行一个 JSON)**:
```jsonc
{"type":"progress","id":"batch-001","completed":1,"total":3,"current_file":"a.ttf","progress":0.33}
{"type":"file_done","id":"batch-001","file":"a.ttf","success":true,"outputs":[{"format":"woff2","path":".../a.subset.woff2","size":12345}]}
{"type":"file_done","id":"batch-001","file":"a.ttf","success":false,"error":"subset failed: ..."}
{"type":"complete","id":"batch-001","ok":2,"fail":1,"elapsed":1.23}
{"type":"error","id":"batch-001","message":"..."}
```

## 实施阶段

### 阶段 1:脚手架 + sidecar 协议(基础)
- `ftl-tauri/` 初始化:Vite + Vue 3 + naive-ui + Pinia
- Tauri v2 初始化:`cargo tauri init`,`tauri.conf.json` 基础配置
- Python sidecar:`sidecar/ftl_worker.py` 实现 stdio NDJSON 主循环,复用 `font/`、`utils/`
- 协议类型:`src/types/messages.ts` 定义 TS 类型
- **验证**:命令行手动给 sidecar 喂 JSON,确认能子集化产出文件

### 阶段 2:Rust 侧 sidecar 桥接
- `src-tauri/src/sidecar.rs`:spawn sidecar 进程、stdin 写入、stdout 逐行解析、转发为 Tauri event
- Tauri command:`start_process`、`cancel_process`,前端通过 `invoke()` 调用
- **验证**:前端发一个 invoke,Rust 起 sidecar 跑通单个文件子集化

### 阶段 3:前端 UI(对应现有功能)
- `FileDropZone.vue`:Tauri `onDragDropEvent` 原生拖拽 + 点击选择(naive-ui Upload)+ 文件列表
- `CharsetInput.vue`:文本域 + 预设 chips(`PresetChips.vue`)+ TXT 导入/去重/清空
- `FormatOptions.vue`:格式复选 + 输出位置单选 + 自定义目录选择(Tauri dialog)
- `AdvancedDialog.vue`:5 个开关(naive-ui Modal)
- `ProcessingDialog.vue`:进度环 + 当前文件 + 结果列表(naive-ui Modal + Progress)
- 主题:naive-ui `darkTheme` + 系统蓝,跟随系统外观
- **验证**:完整跑通端到端,功能与 Flet 版一致

### 阶段 4:跨平台 + 细节
- `useClipboard.ts`:mac(osascript)/ Win(PowerShell `Get-Clipboard`)/ Linux(xclip)
- 跨平台路径处理(`Path` → 各 OS 分隔符)
- 「常用中文」预设修复(内嵌真实常用字表)
- 三端测试
- **验证**:三平台都能拖拽/粘贴/处理

### 阶段 5:打包发布(后续)
- `sidecar/build.sh`:PyInstaller `--onefile` 打包三平台 sidecar
- Tauri `externalBin` 注册 + target-triple 命名
- `cargo tauri build` 产出三平台安装包
- macOS 签名/公证(可选)

## 与 Flet 版的关系
- 全新仓库 `ftl-tauri/`,不动现有 Flet 代码(Flet 版仍在 main 分支可用)
- Python 后端逻辑**直接复用**(`font/`、`utils/` 拷贝到 `sidecar/`),只新增 `ftl_worker.py` 适配 stdio 协议
- 业务逻辑零改动:子集化流程、格式转换、输出目录规则、5 个高级选项映射全部保留
- 顺带修复:「常用中文」预设 bug、死代码(`zip_results`/`generate_timestamp_name`)清理

## 风险与备注
- **sidecar 启动延迟**:每次 spawn Python 进程有冷启动开销(~200-500ms)。优化方向:sidecar 常驻,前端启动时拉起,通过 stdin 持续发指令(协议已按常驻设计,带 `id` 批次号)。
- **PyInstaller 包体**:打包后 sidecar 单文件 ~30-50MB(fontTools + brotli + Python 运行时)。三平台各一份。
- **naive-ui 拖拽限制**:naive-ui 的 Upload 拖拽用 HTML5 API,在 Tauri 里拿不到绝对路径。所以拖拽**必须走 Tauri native `onDragDropEvent`**,Upload 只用于点击选择 + UI 呈现。这点架构上已分开(`useFileDrop.ts` 独立)。
- **Webview 安全**:`tauri.conf.json` 要配 `fs` scope 允许访问字体文件路径;sidecar 是独立进程不受 webview 沙箱限制,文件读写由 Python 直接做(和现在一样)。
- **当前阶段产出**:本计划覆盖阶段 1-4(开发完成、系统 Python 可跑)。阶段 5(打包发布)作为后续单独任务。
